**Step 4: Fix the torch.tensor GAE bug (line 190)**

In `compute_gae()`, replace:
```python
advantages = torch.tensor(advantages, dtype=torch.float32).to(device)
returns = advantages + torch.tensor(values, dtype=torch.float32).to(device)
```

With:
```python
advantages = torch.stack([a.detach() if isinstance(a, torch.Tensor) else torch.tensor(a) for a in advantages]).to(device)
returns = advantages + torch.stack([v.detach() if isinstance(v, torch.Tensor) else torch.tensor(v) for v in values]).to(device)
```

Note: The `advantages` list contains scalar floats (deltas computed from reward + gamma*next_value - value), but `values` contains tensors from model forward pass with `requires_grad=True`. The `.detach()` is needed for `values`. For `advantages`, the items may be tensors if `rewards[t]` or `values[t]` are tensors — use the isinstance check for safety.

**Step 5: Ensure save directory creation**

In the main() function, before the `torch.save()` call (currently line 454-455), add:
```python
from pathlib import Path

```

Similarly for the `plt.savefig()` call (line 450), add:
```python
Path(f'figures/model_performance/').mkdir(parents=True, exist_ok=True)
```

**Important:** The `train_with_rollouts` function references module-scope variables `hidden_dim`, `device`, `reset_memory`, and `bias` — these must be passed as parameters or the function must be moved inside `main()`. The simplest fix: move `train_with_rollouts` inside `main()` as a nested function, or add these as parameters. Choose whichever is less disruptive.

The recommended approach: keep `train_with_rollouts` at module scope but add the missing variables as parameters:
```python
def train_with_rollouts(env, model, optimizer, epoch, n_trials, gamma, rollout_buffer_size, hidden_dim, device, reset_memory, bias, beta_ent=0.0):
```

Then update the call site in `main()` to pass these extra arguments.

**Bug A: NeuroGym state extraction crash (train_multitask.py)**

The state extraction code at lines 481-487 (in `train_epoch_interleaved`), lines 588-594 (in `train_epoch_trial_interleaved`), and lines 676-682 (in `train_epoch_block_interleaved`) accesses PIE-only attributes (`bucket_positions`, `bag_positions`, `helicopter_positions`, `hazard_triggers`) on ALL envs including NeurogymWrapper.

In each of these THREE locations, replace the state extraction block:
```python
states = np.array([
    env.trials,
    env.bucket_positions,
    env.bag_positions,
    env.helicopter_positions,
    env.hazard_triggers,
])
distances = np.abs(states[3] - states[1])
```

With env-type dispatch:
```python
if hasattr(env, 'bucket_positions'):
    # PIE environment — structured state arrays
    states = np.array([
        env.trials,
        env.bucket_positions,
        env.bag_positions,
        env.helicopter_positions,
        env.hazard_triggers,
    ])
    distances = np.abs(states[3] - states[1])  # heli - bucket
else:
    # NeurogymWrapper — trial-based tracking
    n_completed = len(env.trials)
    states = {
        'trials': env.trials,
        'rewards': env.rewards_history,
        'trial_lengths': env.trial_lengths,
    }
    # For NeuroGym, distance metric is not applicable; use zeros
    distances = np.zeros(n_completed) if n_completed > 0 else np.array([0])
```

**Important:** Also update the `results[task_id]` assignment and `self.metrics[task_id].add_epoch()` call after each of these blocks. The `metrics.add_epoch()` receives `distances=` which should work with either format. The `results[task_id]['states']` will be a dict for NeuroGym instead of an ndarray — this is fine for logging purposes.
