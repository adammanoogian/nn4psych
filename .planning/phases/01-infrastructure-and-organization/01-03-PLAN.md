  <files>src/nn4psych/bayesian/numpyro_models.py</files>
  <action>
Fix the critical JAX tracing bug in `compute_normative_model()` (currently around line 149 in the function `step_fn`).

**The bug:** Lines 149-154 use a Python `if context == 'changepoint':` inside `step_fn`, which is called by `jax.lax.scan`. JAX traces `step_fn` once at compile time. The Python `if` evaluates the string comparison at trace time and bakes one branch into the XLA graph. The other branch is silently dead code.

**The fix:**

1. At the top of `compute_normative_model()`, BEFORE the `step_fn` definition, convert the Python string to a JAX boolean:
   ```python
   is_changepoint = jnp.bool_(context == 'changepoint')
   ```

2. Inside `step_fn`, replace the Python if/else (lines 149-154):

   Before:
   ```python
   if context == 'changepoint':
       lr_t = omega_t + tau_prev - (omega_t * tau_prev)
   else:  # oddball
       lr_t = tau_prev - (omega_t * tau_prev)
   ```

   After:
   ```python
   # Use jax.lax.cond for JAX-compatible conditional
   # (Python if/else is evaluated at trace time, not runtime)
   lr_t = jax.lax.cond(
       is_changepoint,
       lambda _: omega_t + tau_prev - (omega_t * tau_prev),   # changepoint: Eq. 2
       lambda _: tau_prev - (omega_t * tau_prev),              # oddball: Eq. 3
       operand=None,
   )
   ```

   Note: `jax.lax.cond(pred, true_fun, false_fun, operand)` — both branches must accept the same operand and return the same shape. Here both return a scalar, so this is satisfied. The `lambda _:` with `operand=None` is the standard pattern when the branches only use closed-over variables.

3. `is_changepoint` is a closed-over variable inside `step_fn` (not iterated by scan). This is correct — `jax.lax.scan` only iterates over the `xs` argument (here `jnp.arange(n_trials)`), not closed-over values.

4. Keep the comments referencing Equations 2 and 3 from Loosen et al. (2023).

5. Remove the now-unused `learning_rate`, `omega`, `tau`, `normative_update` pre-allocated arrays at lines 104-107 — they were never used since `jax.lax.scan` returns the outputs. (They are allocated but overwritten by the scan output.)

**Verify the fix is correct:** Both lambda branches return a single scalar float (lr_t). The `is_changepoint` bool is a JAX tracer-compatible value. `jax.lax.cond` will include BOTH branches in the XLA graph and select at runtime.
