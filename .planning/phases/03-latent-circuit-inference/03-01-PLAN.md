**Part B: Create circuit_inference.py with collect_circuit_data()**

Create src/nn4psych/analysis/circuit_inference.py with the function:

```python
def collect_circuit_data(
    model: ActorCritic,
    env_class,  # SingleContextDecisionMakingWrapper
    modality_contexts: list = [0, 1],
    n_trials_per_context: int = 300,
    max_steps: int = 500,
    device: str = 'cpu',
    env_kwargs: dict = None,
) -> dict:
```

This function:
1. For each modality_context in modality_contexts:
   a. Create a SingleContextDecisionMakingWrapper with that modality_context, set_num_tasks(1)
   b. Run the trained model through n_trials_per_context trials
   c. At EACH timestep, record:
      - u_t: the input state vector = np.concatenate([norm_obs, env.context, [reward]]) — shape (input_dim=7,)
      - z_t: the actor logits from model() — shape (action_dim=3,) — use raw logits, NOT softmax
      - y_t: the hidden state h.squeeze().cpu().numpy() — shape (hidden_dim=64,)
   d. After each trial completes, store the per-timestep arrays

2. Handle variable trial lengths by TRUNCATING all trials to the MINIMUM trial length observed across ALL trials from ALL contexts. This ensures uniform T for LatentNet. Context-DM trials typically all run to max_steps (1000 per Phase 2 notes), but if timing varies, truncation is the safe approach.

3. Stack into tensors:
   - u: shape (n_total_trials, T, 7) — float32
   - z: shape (n_total_trials, T, 3) — float32
   - y: shape (n_total_trials, T, 64) — float32
   Where n_total_trials = len(modality_contexts) * n_trials_per_context

4. Return dict with keys: 'u', 'z', 'y' (numpy arrays), 'T' (int), 'n_trials' (int), 'modality_contexts' (list), 'n_trials_per_context' (int)

Important implementation details:
- Use model.eval() and torch.no_grad() context
- Reset hidden state at the start of each trial block (not between individual trials — maintain continuous hidden state within a modality context block, as during training)
- Actually, reset hidden state per-trial to match extract_behavior_with_hidden pattern: h = model.reset_hidden(batch_size=1, device=device, preset_value=0.0) at the start of each trial
- Record u BEFORE model forward pass (the input that produced this timestep's hidden state)
- Record z and y AFTER model forward pass
- Use argmax for action selection (deterministic, consistent with extract_behavior_with_hidden)

Also add a helper function:
